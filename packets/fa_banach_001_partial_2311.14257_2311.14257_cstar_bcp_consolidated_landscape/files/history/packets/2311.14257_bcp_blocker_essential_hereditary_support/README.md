# BCP Blockers Have Essential Hereditary Support

Status: likely valid partial result; strengthens the essential-blocker bridge.

Source target: arXiv:2311.14257, Question 3.

## Result

Let `A` be a C*-algebra with BCP, and let `C={c_n}` be a countable BCP
blocker from the exact BCP/countable-capture equivalence.  Define a positive
element

```text
h = sum_n 2^{-n-2} (c_n^* c_n + c_n c_n^*) / (1 + ||c_n||^2).
```

Then the hereditary subalgebra

```text
B = cl(h A h)
```

is essential in `A`.

Consequently `A` contains an essential sigma-unital hereditary subalgebra
canonically generated from the BCP blocker.

## Why This Matters

An earlier packet proved only that the closed two-sided ideal generated by a
BCP blocker is essential.  That was not enough: a rank-one projection in
`B(H)` for nonseparable `H` generates the nonseparable ideal `K(H)`.

This new lemma is stronger.  It says the actual hereditary support of the
blocker squares cannot have a nonzero annihilator.  If it did, a unit vector
in that annihilator would be orthogonal on both sides to every `c_n`, giving
exact countable capture and contradicting the blocker property.

## Remaining Limitation

Essential sigma-unital hereditary support still does not by itself imply UBCP:
`B(H)` has essential hereditary subalgebras in trivial ways, and nonseparable
local models can still be too large.  The missing upgrade is to extract a
countable local coefficient model from this essential hereditary support, or
show that failure to do so produces countable capture.

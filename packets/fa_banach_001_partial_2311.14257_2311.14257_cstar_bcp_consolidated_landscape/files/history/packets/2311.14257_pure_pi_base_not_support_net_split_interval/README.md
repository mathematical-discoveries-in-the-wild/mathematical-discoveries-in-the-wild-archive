# Pure-State Pi-Base Does Not Force a Separable Right-Support Net

Status: likely valid counterexample to the intermediate lane-19 support-net
bridge; not a counterexample to the source arXiv:2311.14257 problem.

Source target: arXiv:2311.14257, Question 3, asking for a noncommutative
C*-algebra version of the commutative `C(K)` BCP/UBCP criterion.

## Counterexample

Let `K` be the endpoint-free split interval
`([0,1] x {0,1}) \ {(0,0),(1,1)}` with lexicographic order and the order
topology, and let `A=C(K)`.

Then:

```text
P(A) has a countable pi-base,
A has UBCP,
but A has no separable right-support net.
```

More strongly, no nonzero positive `e in C(K)` has separable principal ideal
`cl(C(K)e)`.

## Why

This standard split interval has a countable pi-base, but every nonempty open
subset contains a copy of a Sorgenfrey interval and therefore is not second
countable. For commutative `A=C(K)`, the closed ideal generated by `e` is
`C_0(coz(e))`. This ideal is separable only when `coz(e)` is second countable.
Since every nonzero `e` has nonempty open cozero set, no nonzero principal
ideal is separable.

The source commutative theorem still gives UBCP because `K` has a countable
pi-base. Thus the previous hoped-for bridge

```text
pure-state countable pi-base => countable separable right-support net
```

is false even for commutative C*-algebras.

## Consequence

The right-support-net theorem remains a useful sufficient condition, but it
cannot be the necessary intermediate invariant for the full classification.
The next route must mimic the commutative scalar bump mechanism more directly,
or use a weaker local coefficient net rather than separable principal ideals.

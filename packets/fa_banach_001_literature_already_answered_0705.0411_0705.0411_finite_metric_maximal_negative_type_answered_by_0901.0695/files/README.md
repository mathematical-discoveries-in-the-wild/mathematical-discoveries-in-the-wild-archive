# Finite-space maximal negative type endpoint

Status: `literature_already_answered`

Source/open-problem paper: Ian Doust and Anthony Weston, *Enhanced negative
type for finite metric trees*, arXiv:0705.0411v2.

Supporting answer paper: Hanfeng Li and Anthony Weston, *Strict
\(p\)-negative type of a metric space*, arXiv:0901.0695v3.

## Identification

In the Introduction and Synopsis of arXiv:0705.0411v2, page 3, Doust and
Weston note that their infinite necklace tree has strict \(1\)-negative type
but no \(p\)-negative type for \(p>1\). They then ask whether the maximal
\(p\)-negative type of a finite metric space can be strict.

Li and Weston answer this directly in arXiv:0901.0695v3. Their abstract says
that refining enhanced negative type leads to the result that the supremal
\(p\)-negative type of a finite metric space cannot be strict. Section 4 is
titled "Supremal \(p\)-negative type of a finite metric space cannot be
strict", and Corollary 4.3 states precisely that the supremal
\(p\)-negative type of a finite metric space cannot be strict.
